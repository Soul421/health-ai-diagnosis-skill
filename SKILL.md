# 健康行业AI落地诊断 Skill

## 一、定位与触发

帮健康行业企业家快速判断AI是否适合自己、怎么落地、多久回本。面向：企业家、数字化负责人、连锁门店运营者。

**触发**：企业AI诊断/落地评估、"适不适合用AI""能省多少钱"、健康行业AI咨询、降本增效/自动化、门店服务诊断/录音豆/合规、连锁门店数字化。

**不适用**：已有成熟AI应用的企业、非健康行业、纯技术咨询。

---

## 二、诊断总流程（6步）

**核心原则**：计算类工作调用脚本执行，LLM负责收集数据和解读结果。

```
Step 1 信息收集 → Step 2 动态评分 → Step 3 ROI分析
     ↓              ↓              ↓
Step 6 门店诊断 ← Step 5 路线图 ← Step 4 场景推荐
```

| 步骤 | 做什么 | 工具/文件 |
|------|--------|----------|
| **1. 信息收集** | 26题分4批+每批确认+最终总表 | `templates/诊断问卷.md` |
| **2. 动态评分** | 5维度评分+行业动态权重 | `scripts/calculate_score.py` |
| **3. ROI分析** | 三档回本+隐性成本+策略对比 | `scripts/calculate_roi.py` |
| **4. 场景推荐** | 10场景动态计算+Top 3排序 | `scripts/calculate_scenarios.py` |
| **5. 路线图** | 三阶段落地+预算+风险预警 | 报告模板 |
| **6. 门店诊断** | 门店/连锁关键词触发，可选 | `references/门店服务诊断方案.md` |

## 三、架构总览（三层API设计）

借鉴 cognee 知识图谱框架的架构思想，采用三层设计：

```
┌─────────────────────────────────────────────────┐
│  顶层 API（4个入口）                              │
│  run_full_diagnosis / run_roi_only              │
│  run_store_diagnose / run_pitfall_check         │
├─────────────────────────────────────────────────┤
│  中层 Pipeline + Task（流程编排）                 │
│  DataValidation → Score → Scenario → ROI → ...  │
│  + 状态持久化 + 数据一致性校验                     │
├─────────────────────────────────────────────────┤
│  底层 引擎脚本 + 本体知识库                        │
│  calculate_*.py  +  ontology.py                 │
└─────────────────────────────────────────────────┘
```

### 3.1 顶层API（4个入口）

调用 `scripts/pipeline.py`，一参入参，直接出结果。

| 入口函数 | 适用场景 | 输出 |
|---------|---------|------|
| `run_full_diagnosis()` | 完整诊断，企业全面评估 | 评分+ROI+场景+路线图 |
| `run_roi_only()` | 快速测算回本周期 | ROI三档测算结果 |
| `run_store_diagnose()` | 连锁门店专项诊断 | 门店场景+服务质量+坑点 |
| `run_pitfall_check()` | 快速排查落地风险 | Top10坑点预警 |

### 3.2 Pipeline 引擎（中层编排）

`scripts/pipeline.py` 核心特性：
- **Task 编排**：6个Task按依赖关系自动执行，支持跳过/回溯
- **状态持久化**：诊断过程可保存为JSON，中断续答（`resume_diagnosis()`）
- **数据校验**：4条合理性规则自动检查数据矛盾，异常时预警
- **Session管理**：每个诊断会话有唯一ID，全流程可追溯

### 3.3 本体知识库（底层领域知识）

`scripts/ontology.py` 结构化领域先验知识，所有脚本统一调用：

| 知识模块 | 内容 | 数量 |
|---------|------|------|
| 行业配置 | 5种行业类型的动态权重配置 | 5类 |
| 维度定义 | 5维度评分体系+结论映射 | 5维 |
| 场景基准 | 10大AI落地场景完整参数 | 10个 |
| ROI参数 | 三档测算+隐性成本系数 | 3档 |
| 合规红线 | 禁用词库+风险等级 | 3类 |
| 坑点清单 | 25个落地坑点（4级严重度） | 25条 |
| 校验规则 | 数据合理性校验规则 | 4条 |
| 路线图 | 三阶段落地模板 | 3阶段 |

### 3.4 专项计算脚本（底层引擎）

计算类工作调用脚本执行，LLM负责收集数据和解读结果。脚本均支持 `--self-test` 自测，输入JSON输出JSON。

| 脚本 | 用途 | 调用时机 |
|------|------|---------|
| `scripts/calculate_score.py` | 五维度评分+4种行业权重适配 | 问卷收集完成后 |
| `scripts/calculate_scenarios.py` | 10场景动态计算+Top3排序 | 痛点排序确认后 |
| `scripts/calculate_roi.py` | 三档回本+隐性成本+策略对比 | 经营数据收集后 |

## 四、关键决策规则

- **行业权重**：5维度（数字化/资源/痛点/团队/合规）按行业动态调整：医疗机构合规25%、纯线上数字化30%、会销团队20%、默认25/20/25/15/15。
- **报告版本**：完整版(10章，深度分析) / 简化版(6章，快速了解)。默认完整版，"简单看看"用简化版。
- **门店触发**：检测到「门店/连锁/录音豆/合规」等关键词主动询问，同意后读门店诊断方案。
- **问卷交互**：26题分4批+每批确认+回溯修改。「跳过」标空，「改第X题」回溯。

## 五、核心坑点 Top 5

> 健康行业AI落地最容易踩的5个坑，完整25条见 `references/坑点清单.md`。

1. **红线词地雷**：AI写文案爱用"治愈""根治""最佳"等禁用词，罚款20万起。必须建敏感词库+人工复核双保险。
2. **员工软抵抗**：表面配合实际不用、故意用错证明AI不行、抱团抵制——找年轻种子员工当突破口，店长必须带头用。
3. **数据质量差**：中老年客户手机号换号率30%+、信息是子女填的。AI项目启动前必须先做数据清洗，脏数据不如不用。
4. **隐性成本冰山**：只算软件费是自欺欺人——数据清洗、内容审核、培训、维护，隐性成本是显性的1倍。首年预算=软件费×2。
5. **内容同质化**：大家都用同一个大模型，文案越来越像，客户审美疲劳。必须建公司专属语料库，AI出稿后人改10%保差异化。
