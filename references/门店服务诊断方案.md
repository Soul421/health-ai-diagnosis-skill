# 录音豆+飞书CLI 健康门店服务诊断方案

---

## 一、方案背景

### 问题现状

健康连锁门店普遍面临以下服务质量难题：

1. **总部看不见门店真实服务情况**：只能靠巡店、神秘顾客，成本高、覆盖低、时效性差
2. **员工话术不规范，合规风险大**：保健品/健康产品容易出现功效承诺，一旦被查后果严重
3. **培训效果难以落地**：晨会讲了很多，但员工实际怎么说的没人知道
4. **流程问题被误认为态度问题**：很多客户投诉本质是流程设计缺陷，却归咎于员工
5. **店长能力差异大**：好店长能带出好团队，普通店长就只能靠运气

### 核心洞察

> 健康行业真正缺的，往往不是某一个超级店长，而是让普通店长也能稳定带出合格服务的系统。

---

## 二、方案定位

**一句话**：把门店里散掉的声音，变成总部能看见、店长能复盘、员工能训练、流程能优化的数据。

### 核心价值

| 维度 | 传统做法 | 录音豆方案 |
|------|---------|-----------|
| 监控方式 | 巡店、神秘顾客 | 自动采样、每日分析 |
| 覆盖范围 | 每月几家店 | 所有门店、每天覆盖 |
| 时效性 | 周报/月报 | 每日日报 |
| 成本 | 高（人力+差旅） | 低（飞书生态内） |
| 员工感受 | 被检查、抵触 | 事后复盘、赋能导向 |

### 设计原则

- **不是监控员工，是赋能门店**
- **不是实时监听，是事后复盘**
- **不是替代店长，是让普通店长也能稳定带出合格服务**

## 三、三大核心场景

### 场景一：团购取货

**痛点描述**：
- 老年人记不清买了什么、多少、去哪取
- 地址改错了没同步到所有通知渠道
- 到店发现少发/错发，情绪上来容易产生冲突
- 员工解释半天，老人还是听不懂

**诊断价值**：
不是员工态度问题，是私域团购通知模板的问题。
问题根源在总部的通知设计，不在门店员工。

**优化方向**：
- 企微群推送**大字版确认卡**（产品名、数量、取货时间、取货方式、地址变更截止时间）
- 取货前一天再次推送提醒
- 门店张贴取货流程大字报
- 高峰期增设"团购取货专用通道"

**数据指标**：
- 每日团购取货咨询次数
- 取货相关投诉率
- 地址错误导致的售后占比

---

### 场景二：产品咨询（合规风险）

**痛点描述**：
- 员工为了促成销售，容易讲过界（"坚持吃肯定有改善"、"这个能治XX病"）
- 形成功效承诺风险，一旦被举报/查处后果严重
- 总部培训讲了很多次，但员工实际怎么说的没人知道
- 新人培训不到位，老员工坏习惯难改

**诊断价值**：
识别合规风险话术，与标准话术库自动比对，发现问题及时纠正。
用真实案例做培训，比抽象讲"注意合规"有效10倍。

**优化方向**：
- 建立**标准话术库**（按产品分类）
- 每日识别风险话术，推送给店长
- 晨会用真实对话片段做案例培训
- 合规话术考试（新人入职必过）

**合规风险分级**：

| 风险等级 | 表现 | 处理方式 |
|---------|------|---------|
| 🔴 高危 | 功效承诺、治愈率、替代药物 | 立即纠正 + 重点培训 |
| 🟡 中危 | 夸大效果、绝对化用词 | 记录 + 晨会提醒 |
| 🟢 低危 | 表述不够专业、不够完整 | 建议优化 |

---

### 场景三：老年人操作困难

**痛点描述**：
- 会员码找不到，翻半天手机
- 验证码收不到，急得团团转
- 小程序不会用，操作步骤记不住
- 员工反复教，还是学不会

**诊断价值**：
不是员工没引导，是流程设计没考虑老年人使用习惯。
问题根源在产品和流程设计，不在员工耐心。

**优化方向**：
- **报手机号积分**（不用找会员码）
- 收银台贴**大字操作卡**（步骤配图）
- 小程序首页会员码入口前置
- 重要操作支持语音引导
- 店员帮操作 vs 让老人自己操作的平衡

**数据指标**：
- 每日老年人操作求助次数
- 操作相关客户抱怨占比
- 会员码/支付操作平均耗时

## 四、三层系统架构

### 第一层：门店服务复盘系统（最易跑通）

**目标**：让店长每天有东西可讲、有案例可训

**运作方式**：
- 每店每天采样1小时（上午团购取货高峰 + 下午产品咨询高峰）
- 每晚自动转写、分类、分析
- 次日早上推送门店服务日报给店长
- 店长拿着系统产出直接带员工开晨会复盘

**输出物**：
- 今日高频问题TOP5
- 需要纠正的话术（合规风险）
- 晨会真实案例推荐（1-2个）
- 服务亮点（正向激励）

**落地周期**：1-2周

---

### 第二层：总部运营优化系统

**目标**：从门店数据中发现系统性问题，优化总部流程

**运作方式**：
- 10家店持续采样2-3周
- 跨门店汇总分析，看趋势和规律
- 发现共性问题，推动总部层面的流程优化

**分析维度**：
- 哪类产品最易被问错
- 哪类话术最容易过界
- 哪个流程最容易造成售后
- 不同门店服务水平差异
- 新老员工话术差异

**改四件事**：
1. **改SOP**：服务流程标准化
2. **改培训课件**：用真实案例更新培训内容
3. **改企微通知模板**：优化通知文案和格式
4. **改产品说明和页面路径**：降低客户理解成本

**落地周期**：1-2月

---

### 第三层：员工主动查询助手

**目标**：员工遇到不确定的问题，主动查标准话术

**运作方式**：
- 在飞书建立话术知识库（按产品/场景分类）
- 员工遇到不确定的问题，主动在飞书搜索
- 知识库返回标准话术和注意事项
- 员工主动调用，不是实时监听

**标准话术结构**：
1. 先说明不能替代药物（合规前置）
2. 产品定位（是什么、不是什么）
3. 慢病/用药人群建议咨询医生
4. 成分和适用场景介绍
5. 真实用户反馈（不夸大）

**为什么更容易被接受**：
- 是员工主动查询，不是被监听
- 是帮员工更好地服务客户，不是抓把柄
- 查询数据可用于优化知识库，形成正向循环

**落地周期**：2-4周

## 五、试点方案（轻量启动）

### 试点规模

| 项目 | 数量 | 说明 |
|------|------|------|
| 试点门店 | 10家 | 覆盖不同区域、不同店长水平 |
| 录音豆数量 | 12台 | 每店1台，重点店2台 |
| 采样场景 | 3个 | 团购取货、产品咨询、售后 |
| 每日采样时长 | 1小时/店 | 上午30分钟+下午30分钟 |
| 试点周期 | 2-4周 | 2周验证可行性，4周验证效果 |

### 验证5件事

1. **能不能稳定采到有效对话**
   - 录音质量是否清晰
   - 每天能否稳定拿到1小时有效对话
   - 设备管理是否方便

2. **转写质量能不能看懂**
   - 飞书妙记的转写准确率
   - 专业术语、产品名能否正确识别
   - 多人对话能否区分说话人

3. **知识库能不能比对出问题**
   - 合规风险话术识别准确率
   - 高频问题分类是否合理
   - 标准话术覆盖度够不够

4. **店长愿不愿意用报告开晨会**
   - 报告好不好懂
   - 晨会案例够不够实用
   - 店长每天花多少时间在这上面

5. **总部能不能从报告里改出具体流程**
   - 能不能发现共性问题
   - 问题能不能归因到流程/培训/产品
   - 改了之后有没有效果

### 落地顺序

```
第1步：先听见门店
   ↓
第2步：再训练员工
   ↓
第3步：再优化流程
   ↓
第4步：最后才是私域自动化
```

**为什么是这个顺序？**
- 先有数据，才能谈优化
- 先帮员工变强，再谈自动化
- 流程问题不解决，自动化只会放大问题

## 六、与店长的关系

### 录音豆不是替代店长，是帮店长变强

| 店长负责 | 系统负责 |
|---------|---------|
| 判断和决策 | 记录和归类 |
| 带人和培训 | 比对和沉淀 |
| 临场应变 | 数据分析 |
| 客户关系维护 | 发现规律和趋势 |

### 总部的角色

总部负责把个别门店的经验，变成所有门店都能用的标准：
- 发现好的做法 → 总结成SOP → 推广到所有门店
- 发现共性问题 → 从总部层面优化 → 减少门店压力
- 发现优秀店长 → 提炼方法论 → 培养更多好店长

### 员工的角色变化

- 从"凭感觉说话" → "按标准话术说话"
- 从"被动接受检查" → "主动查询学习"
- 从"只靠个人经验" → "有系统支持"

## 七、技术实现方案

### 技术栈

| 环节 | 工具 | 说明 |
|------|------|------|
| 录音采集 | 飞书录音豆 | 硬件设备，门店放置 |
| 语音转写 | 飞书妙记 | 自动转写，支持多人对话 |
| 数据拉取 | lark-cli | 命令行工具，批量获取妙记文本 |
| 分析处理 | AI大模型 | 分类、比对、生成报告 |
| 知识库 | 飞书多维表格 | 标准话术库、合规规则库 |
| 报告输出 | 飞书文档 | 每日自动生成门店服务日报 |
| 消息推送 | 飞书机器人 | 推送日报到店长群 |

### 数据流

```
录音豆录音
    ↓
飞书妙记自动转写
    ↓
lark-cli 批量拉取转写文本
    ↓
AI分析处理：
  - 对话分类（场景识别）
  - 合规风险检测
  - 高频问题统计
  - 话术质量评分
    ↓
生成门店服务日报（飞书文档）
    ↓
飞书机器人推送给店长 / 总部运营
```

### 实施周期

- **试点版**：1-2周（单店验证）
- **小规模推广**：2-4周（10家店）
- **全部门店覆盖**：1-2月
- **持续优化**：长期迭代

### 成本估算

| 项目 | 费用 | 说明 |
|------|------|------|
| 飞书录音豆 | 按设备采购 | 一次性投入 |
| 飞书妙记 | 基础版免费 | 高级版按需升级 |
| lark-cli | 免费 | 开源工具 |
| 飞书多维表格 | 基础版免费 | 高级版按需 |
| AI分析 | 按调用量计费 | 可接入自有大模型 |

**基础版接近零成本**（飞书免费版即可跑通核心流程）

## 八、风险与应对

### 风险1：员工抵触（觉得被监控）

**应对**：
- 明确告知是事后复盘，不是实时监听
- 强调是赋能工具，不是抓把柄
- 正向激励为主（表扬好的，不点名批评）
- 让员工参与知识库建设，有归属感

### 风险2：隐私合规问题

**应对**：
- 门店张贴提示："本店服务质量监测，仅用于培训提升"
- 录音数据仅限内部培训使用，不对外泄露
- 定期删除原始录音（保留分析结果即可）
- 涉及客户隐私的内容脱敏处理

### 风险3：转写质量不高

**应对**：
- 选择安静的位置放置录音豆
- 建立专业术语词库，提升识别准确率
- 人工抽查校对，逐步优化
- 重点抓趋势和规律，不纠结100%准确率

### 风险4：店长不用

**应对**：
- 报告设计得尽量简单，5分钟能看完
- 晨会案例直接可用，不用店长自己准备
- 总部运营定期跟进，帮助店长用好
- 设立标杆门店，分享经验

## 九、成功衡量指标

### 短期指标（1个月）
- 门店日报打开率 > 80%
- 晨会使用案例率 > 60%
- 合规风险话术识别准确率 > 70%
- 店长满意度评分 > 4分（5分制）

### 中期指标（3个月）
- 合规风险话术出现率下降 30%
- 高频问题导致的售后下降 20%
- 新员工培训周期缩短 30%
- 客户满意度提升 15%

### 长期指标（6个月）
- 门店服务标准化程度显著提升
- 总部流程优化有数据支撑
- 优秀经验可快速复制到新门店
- 整体运营效率提升 20%+

## 十、总结

健康连锁门店的服务质量问题，本质上是一个**规模化复制**的问题：
- 靠好店长，只能管好少数几家店
- 靠巡店检查，成本高、覆盖低、时效差
- 靠培训讲课，听的时候懂了，做的时候又走样

**录音豆+飞书CLI的方案，提供了第三条路：**

把散落在门店里的声音收集起来，变成可分析、可沉淀、可复制的数据资产。
用系统的力量，让普通店长也能稳定带出合格服务。

先听见门店 → 再训练员工 → 再优化流程 → 最后才是自动化。
